# Decision Rubric

## Use

Choose **Use** when current active work clearly needs it, integration friction is low, evidence is credible, downside is limited, and it improves speed, quality, risk, or cost now.

## Test

Choose **Test** when fit is plausible but unproven and a small pilot can validate it.

Suggested test size: 30–90 minutes, one workflow, one measurable output.

## Watch

Choose **Watch** when it is promising but immature, not needed for current work, pricing/access is unclear, or switching now would distract.

## Skip

Choose **Skip** when there is no active use case, it is mostly hype, duplicates working tools, or risk exceeds value.

## Anti-patterns

- "This is interesting" without a decision.
- "Could be useful" without a use case.
- Long summaries before verdict.
- Recommending adoption because a repo is popular.
- Adding a backlog item when "watch" or "skip" is enough.
